Gravity(A):万有引力公式

Deep Learning
Project
发布日期:   2020-07-16

1. 设想

1.1. 基本设想

训练粒子在引力场中运动

1.2. 目标

1.2.1. 宽线区

  • Single BH BLR云团的动力学
  • 两体运动
  • SMBBH BLR粒子动力学
  • SMBBH Merge 过程BLR粒子动力学

1.2.2. 星系并合

  • 动力学过程
  • 后期再补上能量转化过程

1.2.3. 终极目标

宇宙演化：混沌初开–星系形成

目前只有动力学设想，没有能量转化过程。

就是不晓得要消耗多少资源

2. 开干

2.1. 基本假设

  • 只考虑二维图像（其实三维也可以）
  • 不考虑粒子间相互作用
  • 模型(LSTM)

2.2. 阶段一： 实现粒子的单双势井轨道

2.2.1. 数据生成

  • 单势井
  • 轨道参数

2.2.2. 损失函数

方式一：

单个粒子轨迹仅与下一时刻比较

$$
L_1 = \frac{\sum (\mathbf{\hat{rv}_i^{t+1}} - \mathbf{rv_i^{t}})^2}{\sum (\mathbf{rv_i^{t+1}} - \mathbf{rv_i^{t}})^2}
$$

方式二：

已知某一时刻，粒子的位置$r$，速度$v$和加速度$a$，将损失函数改为$r$和$v$变化与$v$和$a$的余弦相似度,来定义方向。

$$
L_{2} = - \sum (-coss\ similary(r,\hat{r})-coss\ similary(v,\hat{v})+(|r|-|\hat{r}|)^2 + (|v|-|\hat{v}|)^2)
$$

最佳$L_2 = -2$

2.2.3. 模型

简单的全连接层，参数并不多。

model =  tf.keras.Sequential([
    tf.keras.layers.Dense(20, activation='relu'),
    tf.keras.layers.Dense(100, activation='relu'),
    tf.keras.layers.Dense(50, activation='relu'),
    tf.keras.layers.Dense(4)
])

这样一个模型可以模拟多级泰勒展开，应该是最简单的模型了，表现应该比较稳定。

2.2.4. Fitting

2.2.4.1. 测试一 L1

考虑二维轨道平面，输入为[rx,ry,vx,vy]输出为[rx,ry,vx,vy]

效果并不理想，最终单步精度基本稳定在$10^{-5}$的更新步长上，但几乎无法达到与样本相同的精度。而且轨道积分本来要求精度就比较高，而且这个模型的Loss并没有考虑到其中的物理机制，多步积分后将一无是处。

2.2.4.2. 测试二 L2

鉴于上一个Loss的问题，改用余弦相似度。但其中的问题，似乎就变了味道，这样只能说这个model是一个简单的迭代器了，与初忠还有很大差距，属于作弊行为，也许现在知识储备不足，等有能力了再继续按设想的开发。

考虑二维轨道平面，输入为[rx,ry,vx,vy]输出为[vx,vy,ax,ay]

$L_2$最终不断下降，经过50k个epoch后，基本稳定在$8\times10^{-3}$,主要问题在加速度的估计上，这也是变化最大的一项。

result

$L_2$结果:

  • 圆轨道表现不错
  • 偏心率变大后，问题逐渐显现
  • 多步积分后，轨道会变得不稳定，在与训练数据相同更新步长的情况下，最多只能维持半个轨道。
  • 轨道半径比较小的粒子，轨道更不稳定

图就不放了，太丑了。还是需要一个迭代器，来保证精度。

心得

  • Loss 的设置要经过深思熟虑
  • 数据集比较小，模型简单时，数据的读取过程会成为GPU训练速度的瓶颈，cache()会有很大帮助。
  • 余弦相似度，在后期更新过程会略显乏力，在这里体现在速度$v$变化上。
